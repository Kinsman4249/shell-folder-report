## Code of Conduct

This project follows the [Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/), which is the standard code of conduct used by tens of thousands of open-source projects. The full text is available at the link above.

### Summary

In short:
- Be respectful and professional in all project interactions.
- Assume good faith. Disagree about ideas, not people.
- Keep discussions focused on the project - issues, code, documentation, design.
- Make space for contributors of every background and experience level.

### Scope

This Code of Conduct applies to all project spaces: the GitHub repository (issues, pull requests, discussions, code review), and any other channels associated with the project now or in the future.

### Reporting

If you observe behavior that violates this Code of Conduct, please open a private report via the [Security Advisories tab](https://github.com/Kinsman4249/shell-folder-report/security/advisories/new) of the repository. Reports are read by the project maintainer only and handled confidentially.
When reporting, include:
- What happened (links, quotes, screenshots if helpful).
- When and where it happened.
- Anyone else involved or affected.
You will receive an acknowledgment, and the maintainer will follow up with next steps. The maintainer may take any action they deem appropriate, up to and including a permanent ban from the project.

### Maintainer's note

This is a small project. The intent of this document is to set a clear, friendly baseline for how we work together, not to be a legal instrument. If something feels off, raise it. We would rather talk it through than ignore it.
